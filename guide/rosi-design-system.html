ROSI Design System

Functional minimalism — a mission control terminal, dense with information, calm in color. Use the palette switcher in the header to compare all five directions against the same component demos. Default is Direction A (Coherent Verdant), the recommended aligned palette.

Functional Minimal Calm Dense Dark Only 5-Way Compare
✦
5 directions, 1 comparison surface. All component demos (buttons, badges, surfaces, status) re-render live when you pick a palette. The structure is identical across directions so the choice is honest: only the colors move.
!
Stub palette: D (Mint Cyan) or E (Warm Amber). These directions are defined for accent and base only; full surface, text, and status token sets are pending the v3 doc update. Buttons, badges, and accents render with the new color; the rest of the system uses the default Coherent Verdant values until the gap is filled. Pick A, B, or C for a complete comparison.

Why these 5 directions

The five palettes are not five random options. They tell a story about where the design system came from, where it is now, and where it could go. The switcher is a comparison tool, not a menu of finished options.

  1. A
    Coherent Verdant

    The recommendation. A calibrated middle ground between the v0.2 sage family and the dev team's current phosphor. New accent #3FB286 on a green-warm dark base #0F1411. AAA contrast on primary text and accent pairings.

  2. B
    Lean Implementation

    What the dev team is actually shipping right now. Vibrant green #24D18F on a near-black base. The implementation drifted from the v0.2 documentation.

  3. C
    Lean Documentation

    What the v0.2 documentation says we ship. Sage #8AB4A0 on blue-gray #1A1E21. The documentation and the implementation never agreed.

  4. D
    Mint Cyan stub

    A cool departure. Mint #4DC9C9 on a near-black teal base. Use this direction if Coherent Verdant reads too warm for the workspace. Full token set pending the v3 doc update.

  5. E
    Warm Amber stub

    A warm departure. Amber #E0A659 on a warm dark base. Use this direction if Coherent Verdant reads too cool. Full token set pending the v3 doc update.

The real choice is between A, D, and E. B and C are included so the drift is visible.

Accent — Active Palette

4 tokens

A single accent family. Calm enough for sustained reading, strong enough for action signals. Muted is for subtle backgrounds. Strong is the send button and active indicators. One accent, zero confusion. Values below update with the active palette.

--accent-hover — Hover state on accent Brighter. Invites interaction.

Token Naming Convention

Flat, semantic naming. Category first, then variant, then state. No nesting. Machine-readable and human-scannable.

Pattern:
--{category}-{variant}
Examples:
--surface-deepest
--text-secondary
--accent-strong
--sp-4
--radius-md
--motion-base
--z-browser-layer

Design Rules

Hard constraints from the community. Break them only with explicit justification.

Do

Keep it dark — no light theme. Use green accent for interactive/live elements only. Left-align sidebar text. Use uppercase + letter-spacing for section labels. Create depth by tonal layering, not shadows.

Don't

Add colors beyond green accent without functional reason. Use shadows except for overlaying panels and dropdowns. Center-align body text. Use rounded corners larger than 12px except circles. Decorate without purpose.
